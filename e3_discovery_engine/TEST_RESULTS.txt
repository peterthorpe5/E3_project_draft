External DIAMOND compatibility changes in this release
------------------------------------------------------
The DeepClust parser now accepts DIAMOND's documented headerless two-column
cluster membership output and recognised header variants. DeepClust no longer
requests a version-dependent clustering header. Realignment requests an
explicit simple header and uses the configured masking mode. Header-only
realignment output is retained as a valid zero-row Parquet table with a warning.
The external command path still requires confirmation in the user's DIAMOND
environment.
